Casos de Prueba Backend

Los siguientes casos de prueba para el backend se centran en la lógica de negocio, la seguridad y el rendimiento de la aplicación desde la perspectiva del servidor. Algunos aspectos importantes son

Pruebas y Dependencia para la entidad Departamento

Nombre del Método Descripción Resultado Esperado Dependencia
testCreateDepartamento() Prueba la creación de un nuevo departamento. El número de departamentos aumenta en 1 después de crear uno nuevo. Spring Boot Test
testUpdateDepartamento() Prueba la actualización de un departamento existente. El departamento se actualiza correctamente con los datos modificados. Spring Boot Test
testGetDepartamentoById() Prueba obtener un departamento por su ID. Se obtiene el departamento correcto con el ID especificado. Spring Boot Test
testDeleteDepartamento() Prueba borrar un departamento existente. El número de departamentos disminuye en 1 después de borrar uno existente. Spring Boot Test
